### Введение

Целью данного тест-плана является проверка корректности работы CRM-платформы, в частности модулей "Задачи", "Карточка задачи", "Отчёты", главной страницы и связанных с ними элементов интерфейса.

**Объект тестирования:** веб-платформа CRM, доступная по адресу демо-сервера.

**Предполагается провести:**

Для отдельных полей:

- Позитивное тестирование (корректные шаги и данные);
- Негативное тестирование (ввод некорректных данных);

Для всей системы:

- Функциональное тестирование;
- Кросс-браузерное тестирование;
- Юзабилити-тестирование;
- Тестирование интерфейса и элементов управления;

### Тестируемый функционал сайта

Будет протестирована следующая функциональность:

- Отображение главной страницы и виджетов;
- Переход по страницам через боковое меню и виджеты;
- Форма создания задачи;
- Функциональность карточки задачи: статусы, вложения, напоминания;
- Фильтрация и сортировка задач;
- Работа кнопок "Создать задачу", "Создать черновик", "Мои задачи (0)", "Наблюдаемые (0)", “Расширенный поиск”, “Отправить”;
- Отображение и фильтрация данных на странице "Задачи - Отчет";
- График задач по времени и количеству;
- Переключение языка интерфейса.
- Работа со статусами задач (Черновик, В работе, Готово и т.д.);
- Прикрепление файлов и фото;
- Установка напоминания (колокольчик);
- Ведение комментариев и истории изменений;
- Управление чек-листом и прогресс-баром.

### Типы тестирования и используемые инструменты

Для охвата функциональности будут использоваться:

**Функциональное тестирование**

Проверяется соответствие системы заявленным требованиям:

•	Навигация по разделам (главная, задачи, отчёты);

•	Работа кнопок, виджетов, чек-листов;

•	Создание, редактирование и удаление задач;

•	Фильтрация, сортировка и поиск;

•	Смена языка интерфейса;

•	Переходы между статусами задач;

•	Проверка валидности форм (в том числе негативные сценарии).

**Инструменты:** ручное тестирование, DevTools.

**Нефункциональное тестирование**

Оценивается удобство, производительность, кросс-платформенность:

•	**Юзабилити:** логичность интерфейса, доступность элементов, понятность навигации;

•	**Кросс-браузерность:** Chrome, Firefox (разные ОС);

•	**Производительность (визуально):** отклик элементов, скорость загрузки;

•	**Отзывчивость интерфейса:** как элементы адаптируются под разные экраны.

**Инструменты:** Chrome DevTools (вкладка Performance)

**Тестирование мобильной версии**

Проверка основных сценариев на эмулируемых устройствах:

•	Открытие сайта в мобильной версии (через DevTools → Toggle Device Toolbar);

•	Кликабельность элементов, доступность меню, правильность отображения текста.

**Инструменты:** Chrome DevTools.

### Критерии начала и окончания тестирования

**Начало тестирования:**

1. Доступна стабильная тестовая сборка.
2. Подготовлена тестовая среда.
3. Завершена разработка основных модулей.

**Окончание тестирования:**

1. Все запланированные тесты выполнены;
2. Все баги приоритетов Critical/High исправлены;
3. Все баг-репорты оформлены и переданы;
4. Отчёт по тестированию согласован.

### Риски и ограничения

- Ограниченный доступ к серверу может повлиять на выполнение тестов;
- Отсутствие доступа к мобильной версии (тестируется через эмуляцию);
- Возможны баги, связанные с отображением на разных разрешениях экрана (особенно при масштабировании интерфейса в macOS).
